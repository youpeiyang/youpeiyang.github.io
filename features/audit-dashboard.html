审计首页概览

审计看板 (Audit Dashboard)

此看板是整个数字员工（AI Agent）健康状况与企业安全风险等级的核心监测站。作为 IT 运维人员和安全合规官员登入系统后的“第一落脚点”，它提供了高度聚合的宏观视角，协助团队掌控大中型智能化运营现状。

🌟 核心价值与功能亮点

在接入大批量的内部机器人应用或自动任务管线后，每天产生的成千上万条对话（Sessions）往往让安全审计变得像大海捞针一样困难。通过全局审计仪表盘，您可以毫不费力地：

  1. 掌握资源活跃度：通过即刻刷新大盘，秒级掌控当下数字员工的并发请求数有多少；
  2. 洞察告警分布模式：直观区分各种“疑似高危”行为占比情况；
  3. 定位热点问题与来源：分析哪位终端调用者和哪个具体工具导致的问题（报错或超额度）发生频率最高。

📸 看板核心构件详解 (What You See)

登入系统面板后，不同维度的组件展示了数字员工各个阶段的表现形式：

1. 核心指标总览 (Key Metrics Overview)

这块区域居于顶部最显眼的位置，通常覆盖 今日 / 本周 / 本月 的核心数据对比指标：

  • 会话总数 (Total Sessions)：系统当前承载的所有请求或任务对话实例。
  • 活跃会话数 (Active Sessions)：目前仍处于活动或挂起中未结束的主动交流链路。
  • 独立终端接入数 (Unique Endpoints/Clients)：多少独立用户与终端 ID 接入触发了这些会话调用。

最佳实践 (Best Practice) 👉 该部分数据若某天突现“火箭式飙升”，运维团队应当结合后端的报错率去检查是否遭到了恶意脚本反复请求，或是 Agent 本身设计出现了“死循环重试”的 BUG。

2. 风险会话占比分析 (Risk Distribution)

数字员工难免因用户意图错误、API 配额不足或系统故障而出现异常阻塞乃至尝试执行系统底层命令。这些通过“风险打分模型”筛选的记录会出现在这里：

  • 饼图 / 环形图反映比例：清晰显示高危操作（如试图执行敏感 Bash 脚本且被探针捕获拦截）、中等警告、与低风险报错（如 API Error / Max Token 排空）这几类典型异常问题等占所有会话的权重比例分配。
  • 健康度打分体系：帮助安全专员识别系统的环境防污能力和边界防护是不是存在隐患漏洞，防微杜渐，扼杀风险于摇篮。

3. 热点分析：工具与用户词云排行榜

当有上百种外部接口与本地脚本工具供大语言模型去选用时，您可以通过该模块进行快速检阅：

  • 高频内部用户 / Agent调用方：显示究竟是哪些具体的使用者引发了全天绝大多数的操作请求；
  • 高风险工具命中榜：识别哪一类的 Tool（如特定文件读写 fs.write、特定的网络搜刮请求等）伴随的风险失败和拦截次数远超平均水准。

🔎 下一步该怎么做？

如果在此审计面板发现了异常峰值（如某一小时内的安全拦截数暴增），您可以立刻切换跳转至 会话审计与溯源追踪详情 (Session Tracing)，在全链路检索内过滤出对应的时间段和动作标签，精确定位触发问题的罪魁祸首！
